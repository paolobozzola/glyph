# Glyph — v2.0 Milestone Plan

Build order, scope, and acceptance criteria for v2.0. Themes were chosen with the owner
(see `PLAN.md` §v2.0). **AI assist is excluded by design** to preserve the "fully offline —
nothing leaves your Mac" principle. Architecture context lives in `CLAUDE.md` and `SHELL.md`.

Sequencing principle: ship visible richness early (much of it is already bundled in Milkdown
Crepe), then workflow, then the standout macOS integrations. Each milestone is independently
shippable; Sparkle slots in whenever the appcast host exists.

---

## V2.1 — Richer content (Theme A)

Most of this is *enabling and theming* features Crepe already bundles, so it's high
value-to-effort and stays plain Markdown on disk.

- **Syntax-highlighted code blocks + language picker**
  - Enable `Crepe.Feature.CodeMirror`; import its theme CSS (`@milkdown/crepe/theme/common/code-mirror.css`).
  - Verify round-trip: fenced blocks keep their info string (` ```swift `).
  - *Accept:* a `swift`/`js`/`json` block highlights; changing the language persists to Markdown.
- **LaTeX math**
  - Enable `Crepe.Feature.Latex` (+ `latex.css`); inline `$…$` and block `$$…$$`.
  - Serializer must round-trip math as plain `$…$` (remark-math style).
  - *Accept:* type `$E=mc^2$` → renders; save → file contains `$E=mc^2$`.
- **Slash `/` command menu**
  - Enable `Crepe.Feature.BlockEdit`; confirm it lists headings, lists, table, code, quote, hr.
  - *Accept:* on an empty line, `/` opens the menu; choosing "Table" inserts one.
- **Diagrams (Mermaid) + footnotes**
  - Mermaid: a code-block language `mermaid` rendered to SVG (offline lib bundled). Footnotes via `remark-gfm`/`remark-footnotes`.
  - *Accept:* a ```` ```mermaid ```` block renders a diagram; `[^1]` renders a footnote, round-trips.
- **Risk:** confirm these features don't fight our serializer config (`bullet:-`, `rule:-`) or the
  custom paste/image handlers. Re-run the round-trip corpus.

## V2.2 — Faster workflow (Theme B)

- **Command palette (⌘K)**
  - In-bundle overlay (same pattern as the cheat sheet / find bar): fuzzy list of all
    commands *and* document headings; Enter runs/jumps. Bridge command `palette`.
  - *Accept:* ⌘K → type "table" → runs Insert Table; type a heading → jumps to it.
- **Document templates**
  - Templates stored as `.md` files in Application Support (`~/Library/Application Support/Glyph/Templates/`),
    seeded with a couple of defaults. **File ▸ New from Template ▸ …** populates a new doc.
  - *Accept:* "Meeting note" template opens a new doc with its frontmatter + skeleton.
- **Smart paste**
  - Extend the existing capture-phase paste handler: URL pasted over a selection → wrap as a
    link; HTML clipboard → convert to Markdown (bundle a small HTML→MD converter).
  - *Accept:* select a word, paste a URL → `[word](url)`; paste rich HTML → clean Markdown.
- **Table row/column controls**
  - Surface Crepe's table tools (add/remove row/col, alignment).
  - *Accept:* hovering a table shows controls; add-column works and round-trips.

## V2.3 — Rendered-Markdown customization

- **Editor themes & fonts**
  - A small Settings window (or menu) to pick a theme (e.g. Paper / Nord / High-contrast) and a
    body font + size; persisted in `UserDefaults`, pushed to the bundle like `setTheme`.
  - *Accept:* switching theme restyles the editor live and survives relaunch.
- **Custom CSS**
  - Optional user stylesheet (`~/Library/Application Support/Glyph/custom.css`) injected into the
    editor; shared with HTML/PDF export and (where feasible) the Quick Look preview for consistency.
  - *Accept:* adding a CSS rule changes the editor and the exported HTML.
- **Reading / split mode** *(stretch)*
  - Side-by-side source ⇄ rendered, and a chromeless reading view.

## V2.4 — macOS integration (Theme C) — the "only-on-Mac" headline

- **Spotlight content search**
  - `.md` conforms to `public.plain-text` so Spotlight already indexes raw content; add a
    lightweight importer/attributes so titles (frontmatter `title`/first H1) surface nicely.
  - *Accept:* searching a phrase inside a `.md` finds it; the result shows the doc title.
- **Shortcuts.app / App Intents**
  - `AppIntents` for **New Note**, **Append to Document**, **Export to PDF/HTML**.
  - *Accept:* a Shortcut "New Glyph note with text" creates and saves a `.md`.
- **System Services + menu-bar quick capture**
  - An `NSServices` provider ("New Glyph Document from Selection") and an optional `NSStatusItem`
    menu-bar jotter that appends/creates.
  - *Accept:* selecting text in any app → Services ▸ New Glyph Document creates a doc with it.
- **Parked — iCloud / Handoff:** requires App Sandbox, which conflicts with the current
  non-sandboxed direct-download model. Decide as a separate architecture track if pursued.

## V2.5 — Sparkle auto-updates

- Add the Sparkle framework (SwiftPM), generate an EdDSA key pair (public key in the app),
  host `appcast.xml` + DMGs (GitHub Releases). Extend `make dist` to sign the update and
  update the appcast.
- *Accept:* an older build detects a newer release and updates itself; the update is signature-verified.
- Prereq: a stable hosting URL for the appcast (decision needed).

---

## Cross-cutting

- **Round-trip safety:** every content feature (V2.1) must preserve the *file as source of
  truth*. Extend `tests/corpus/` with math, code-with-language, footnotes, and a Mermaid block;
  verify save is deterministic (Playwright against the built bundle, as in M2).
- **Offline:** no feature may make a network request at runtime (AI excluded). Bundle any libs
  (Mermaid, HTML→MD) into the offline `app://` payload.
- **Shortcuts:** keep the letters/digits-only accelerator policy (see `AppDelegate.addCommand`)
  so v2.0 additions work on international keyboards.

## Suggested order

V2.1 (quick, visible) → V2.2 → V2.3 → V2.4 (headline) → V2.5 (whenever the appcast host is ready).
